Usage Guidelines

WordPress Integration

This design system integrates seamlessly with WordPress and Bootstrap 5:

  • Use Bootstrap classes for standard layouts and utilities
  • Use our custom classes for brand-specific styling
  • Follow the modular CSS architecture for new components

File Structure

assets/css/ ├── system/ │ ├── variables.css # Design tokens │ ├── typography.css # Typography system │ ├── layout.css # Layout utilities │ └── utilities.css # Helper classes ├── components/ │ ├── buttons.css # Button components │ └── cards.css # Card components ├── base.css # Main import file └── [template].css # Template-specific styles

Best Practices

  • Use CSS custom properties - Always use our design tokens instead of hardcoded values
  • Follow the cascade - Avoid !important unless absolutely necessary
  • Mobile first - Write mobile styles first, then enhance for larger screens
  • Semantic HTML - Use proper HTML elements for accessibility
  • Component-based - Create reusable components rather than page-specific styles
